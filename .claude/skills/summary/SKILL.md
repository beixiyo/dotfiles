# 任务交接摘要

根据当前对话上下文、代码修改记录和任务目标，写一份**可直接交给下一个 LLM 续写**的交接文档，写入 handoff.md，名字如果冲突你自己改。新 LLM 应能据此理解背景、现状和下一步，无需重读整段对话

## 输出格式
严格按下列章节输出；**没有内容时写「无」或简要说明，不要只写章节标题**

### 背景与目标
- 当前任务因何而起、要解决什么问题
- 本阶段希望达成的具体目标（可拆成 1～3 条）

### 当前进度与现状
- **已完成**：已做完的步骤或已合入的改动
- **进行中**：当前正在改的文件/逻辑，以及卡在哪里（如有）
- **异常说明**：若存在编译失败、运行报错、逻辑错误，必须写明现象、可能原因和复现方式（或「尚未复现」）

### 重要文件与引用
- 与任务强相关的核心文件路径（建议 3～8 个）
- 每个文件一句话说明：在本次任务里扮演什么角色、为何重要

### 已尝试过的方式（可选）
**仅当「确实尝试过某些做法并有结论」时填写本段；若没有尝试过或没有失败/成功结论，直接写「无」即可**

- 若填写：按「做法 → 结果 → 原因/结论」写清，例如：
  - 「用方案 A 做了 X，结果报错 Y，判断是 Z 导致，已放弃/已改用方案 B」
  - 「试过从接口 B 取数据，发现字段不全，改回用接口 A」
- 避免只写「尝试过的方式（如果有）」当内容；没有就写「无」

### 待办与建议下一步（可选）
- **待办**：尚未完成的具体事项（可选）
- **建议给新 LLM 的指令**：接下来应执行的 1～3 条具体操作或验证步骤，便于新对话直接开工（可选）

---

若任务涉及环境、分支、密钥等，在「当前进度与现状」或单独一句补充说明，方便新 LLM 不踩坑
