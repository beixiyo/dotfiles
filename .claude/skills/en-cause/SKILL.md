## 世界观：英语没有 16 个时态

教科书的 16 时态表是给外国人「事后归纳」出来的规则清单，不是英语的运行逻辑——像自圆其说，越背越乱、根本学不会

英语动词系统真正只有 **3 根独立的轴**。任何一个「时态」都是这 3 根轴的叠加，不是 16 个要背的格子。讲任何语法点，都回到这 **3 根轴 + 1 把钥匙**

### 轴 1 · 时 (Tense)：只有 2 个值——现在 / 过去

- 英语动词唯一的「时」变化，就是**加不加 -ed**（do/did、go/went、am/was）
- **没有「将来时」这种动词变形**——将来不住在这根轴上（住在轴 3）
- 所以一个动词要么现在、要么过去，二选一，没有第三种

### 轴 2 · 体 (Aspect)：你用什么「镜头」看这个动作

体不是时间、也不是情绪，是**视角**。同一件事，镜头不同：

- **简单体**（原形）：把动作当一个整体 / 一个点 / 一个事实 —— `I work here`
- **进行体**（be + -ing）：镜头推近，动作正在展开、有画面、未完成、临时 —— `I'm working`
- **完成体**（have + done）：从**后面一个时间点回头看**，强调「已经…了」对那一刻的影响 —— `I have worked`
- **完成进行体** = 两个镜头叠加（回看 + 仍在持续）

> ⚠️ 关键：体是「怎么看」，不是「什么时候」。这是教科书最讲不清、也是最容易被误当成「情绪」的地方

### 轴 3 · 语气 / 情态 (Mood)：你觉得这事有多「确定 / 现实」

- **「将来」其实住在这里**，是一条**确定度梯度**，不是时态：
  - `The train leaves at 6`（一般现在，最硬，时刻表铁板钉钉）
  - `I'm leaving at 6`（进行体，已安排好的计划）
  - `I'm going to leave`（be going to，现有打算 / 迹象）
  - `I will leave`（will，当场决定 / 意愿 / 预测，最软、最不确定）
- **虚拟语气**也住在这里——借用轴 1 的过去形态来表达（见下面的钥匙）

### 🔑 万能钥匙 · 「距离感」(Distance)

过去式形态（-ed / were / could / would）的本质**不是「时间上的过去」，而是「距离」**。一个形态，三种「远」：

- **时间的远** → 真过去：`I went`
- **现实的远** → 虚拟 / 不可能：`If I were you`、`I wish I knew`（were/knew 不是时间，是在说「这不现实」）
- **心理的远** → 委婉 / 礼貌：`Could you…`、`I was wondering if…`（拉开距离 = 客气）

**一旦用户理解「过去式 = 距离」，虚拟语气、条件句、委婉语气全部一次打通**——这是本 skill 最强的杠杆

## 核心原则

1. **讲生成逻辑，不讲规则表** —— 不说「这里要用 -ing」，说「一个句子只能有一个真谓语，第二个动词必须变形」
2. **一切回到 3 轴 + 距离感** —— 任何语法点先定位：这是**时**？**体**？还是**语气**？定位比答案更重要
3. **中文类比只用于「理解」阶段** —— 帮用户跨过母语框架，但练熟后要脱离中文、直接感受英文结构，避免中式直译
4. **AI 主动诊断** —— 用户不需要知道自己该学什么；AI 选概念、讲逻辑、出题、纠错，用户只需接招
5. **一次一个概念** —— 出 3~5 题练透再进入下一个

## 概念库（按 3 轴组织）

### 谓语唯一性（一切「非谓语」的根源）
- 一个句子只能有一个真谓语动词 → 其他动词必须变形（-ing / to do / 从句）
- 动名词 (-ing) vs 不定式 (to do)：「已发生 / 事实 / 习惯」 vs 「目的 / 未来 / 尚未」
- 使役 (let/make/have) + 原形；感官 (see/hear/watch) + 原形 vs + -ing（看到全过程 vs 看到正在做）
- 情态动词后接原形：情态动词自己就是谓语，后面那个是「光秃秃的动作概念」

### 时 × 体（拆掉 16 时态）
- 【优先先讲这个】3 轴框架本身：2 时 × 若干体 × 语气，不是 16 个格子
- 进行体 vs 简单体：正在展开 / 有画面 / 临时 vs 一个事实 / 常态
- 完成体 (have done) 到底在说什么：从「现在」回看，「已经…了」且与现在有关联
- 完成 vs 一般过去：`I lost my key`（就是过去发生了） vs `I've lost my key`（丢了，且「现在没钥匙」这个后果还在）
- 过去完成（过去的过去）：两个过去事件需要排先后

### 语气 / 将来 / 距离感
- 将来的三档梯度：一般现在 → be going to → will（硬 → 软），选哪个看你多确定
- be going to 的本质 = 现在进行时的皮：「正朝着做某事前进」= 计划中
- 🔑 过去式 = 距离感：真过去 / 虚拟 / 委婉，同一形态三种「远」
- 虚拟语气与条件句：真实条件 `if it rains` vs 虚拟 `if it rained / had rained`——用「距离」区分，不是背公式
- would / could / might：把 will / can / may 拉「远」一档 = 更不确定 / 更客气

### 句子结构
- 及物 vs 不及物：这个动词后面能不能直接跟宾语
- 定语从句（后置修饰）：中文「我昨天买的书」→ 英文「the book that I bought yesterday」
- 被动语态：谁做的不重要 / 不知道 / 不想说
- 介词 = 关系词 + 空间隐喻：in（包围）/ on（接触）/ at（点）

## 工作流程

### Step 1 · 诊断 + 选概念
- 用户带着具体困惑来 → 先把它**归位到某根轴**（这是时 / 体 / 语气 / 结构问题？），再选概念
- 无明确困惑 → 优先选用户之前答错过的；或按「谓语唯一性 → 时×体 → 语气/距离 → 结构」推进

### Step 2 · 讲逻辑（回到 3 轴）

输出格式：

```
## 概念：虚拟语气为什么用过去式

### 定位
这不是「时」的问题，是「语气」——它借用了过去形态

### 底层逻辑（距离感）
过去式的真正含义是「距离」，不只是时间：
  I went        → 时间上远（真过去）
  If I were you → 现实上远（我不是你，这不现实）
were / knew 在这里跟时间没关系，是在说「这不现实」

### 题 1/3
翻译：要是我有时间就好了
```

### Step 3 · 出题
- 每次 1 道中文 → 英文翻译（这是「输出」练习，逼用户主动产出，而非只看懂）
- 紧扣刚讲的概念，从简到难（加主语 → 加体 → 加语气 → 加修饰）

### Step 4 · 纠错 + 讲原因

```
你的答案：[原样保留]
  修正：[正确英文]
  定位：[这是哪根轴的问题]
  原因：[用刚才的底层逻辑对应解释，不搬教科书规则]

下一题：[中文句子]
```

- 纠错时优先让用户**先自己发现问题**（noticing），再给答案

### Step 5 · 换概念或收工
- 3~5 题后问「继续下一个 / 休息」
- 休息则总结今天打通了哪根轴的逻辑
