# 写作

根据题目与目的自由组合手法，产出让读者「学到东西 / 被冒犯 / 笑出声 / 猜不透」的文字
不是每篇都要用遍所有手法——**主手法永远不超过 2 个**，其余做支撑

## 第零条：反 AI 味（最高优先级）

AI 味的本质是「所有人的平均味道」。以下特征出现一条就重写：

- **恒温**：情感永远二十六度，不烫手也不冰凉。好文字要有温度突变
- **总分总**：首先/其次/最后/综上所述。像工作汇报的结构就是死刑
- **宏大废话开头**：「在当今 XX 时代」等于走进酒吧先说「作为一个人类」
- **比喻过近**：双刃剑、一面镜子。好比喻让人先愣一下再「哦」，坏比喻直接「哦」然后忘掉
- **均匀无焦点**：每段一样长、每句一样重要、灯光均匀的房间。不均匀才是信息
- **每段收工整**：读者能看出每一处设计意图，就输了。留缺陷、敢跑题、敢戛然而止
- **解释自己的手法**：写完讽刺再解释哪里讽刺，等于自首

情绪的正确写法：**不写形容词，只给事实**
不写「我很痛苦」，写「我当时心率起码在二百以上」「快三十年了，我现在打这些字手都是抖的」

## 第一步：题目诊断

动笔前回答三个问题：

1. **读者带走什么**：学到硬知识 / 被冒犯到 / 笑出声 / 眼前一亮 / 猜不透真假
2. **情绪基调**：展示成果 / 辛辣讽刺 / 冷静论证 / 深情 / 一本正经地胡闹
3. **真假光谱**：全真 → 事实真态度假 → 真假互保 → 全假但自洽。落在哪一格

然后从手法库选主手法

## 好文章的公约数

手法之上有四条铁律，违背任何一条，手法再花哨也是废文：

1. **人的处境在场**：主角可以是人、动物、框架、调味料，但必须影射或映照真实的人——影射型（Markdown 皇帝 = 一类开发者）、镜子型（超市番茄照出「指标达标但没人爱」）、陪衬型（讲鸭讲着讲着，养鸭人的人生从缝隙里漏出来）。检验：把对象还原成它影射的人或行为，文章依然成立；还原后空无一物的拟人只是贴纸（台风「卒」）。纯粹的智力游戏、推导炫技没有体温
2. **靶子真实**：批判的行为、关切的痛点必须真实存在（Spec 迷信、AI 味、粗暴的家庭教育）。没有真靶子的荒诞只是抖机灵
3. **故事和比喻承载论证**：读者跟着人物走完弧线，论点自己长出来。公式和列表只能做佐证，不能做骨架
4. **留下可转述之物**：一句黑话（删不删 src）、一个意象（椅子是热的）、一个数字（心率二百）。读者复述给别人时用的那句话，动笔前就要想好

情绪必须有来源——痛有出处、怒有对象、冷静有立场，否则是情绪表演

## 手法库

### 结构类

- **成果前置**：开头直接列「阅读本文你将学会：X、Y、Z」，量化承诺（「200 行代码」「5 分钟」「十几种开箱即用」）。技术文章默认起手式
- **母比喻系统**：整篇只用一个母比喻，让它长出论证网。味精 → 中央空调 → 超市番茄 → 可乐甜度 → 陶瓷碗，每个支比喻都为主论点添砖。金句从母比喻里长出来（「椅子永远是冷的」）
- **寓言一喻到底**：「我讲个故事就很清楚了」，然后整篇装进一个寓言（Markdown 皇帝的帝国兴衰）。术语混搭制造荒诞感（CAP 定理 × 女频小说 × 言出法随），结尾必须有反转刀
- **自指结构**：文章本身是自己论点的证据。「你正在读的这段文字，是 AI 写的」——莫比乌斯环，翻不到反面。开头埋的话结尾原样收回，含义已变
- **戏仿嫁接**：把经典文本整段移植到新对象上（孔乙己 × React 团队），人物、对白、气氛全保留，只换宾语。文本越严丝合缝，对象错位越荒诞

### 质感类

- **具体性暴力**：细节精确到远超必要。「五千万只鸭」「最重的个体 14.7 斤」「0.1 元五天涨到 5.8 元」「一千四百二十一个小孔，数了三遍」。经历越传奇，数字越要扎实——**传奇与精确互相担保，这就是真假难辨的引擎**
- **生理事实替代情绪**：见第零条。痛苦、愤怒、狂喜全部换算成心率、手抖、一言不发地望着火变小
- **缺陷式文风**：故意的跑题、口语、重复、不收尾。跑题之后要能回得来（或干脆不回，用「好，字数到了」切断）
- **破坏性节奏**：把交易和生活直接砌进正文。「一个赞打一个字」「我先去吃饭了。好，我吃饭回来了」「有人看再写」。第四面墙不是打破，是当承重墙用

### 态度类

- **形式内容错位**：学术腔与网文腔无缝切换——寓言里塞 CAP 定理，脏话嘲讽里给 RAG 原理拆解。错位只是外壳，壳里必须有真论点和真的人，纯炫技的推导游戏不算
- **反串不点破**：全程正面陈述，句句是刀，绝不揭穿自己。「成年人的成熟，就是精心饲养一个可以怪罪的东西，并按月领取饲养费」。留一两个可攻击点维持暧昧，让读者吵起来
- **发明术语与判据**：造新概念 + 给粗暴好用的鉴别标准。「Vibe Specification vs Vibe Coding——就看你会不会频繁删除 src」。读者获得一句可以转述的黑话
- **名言错位改写**：拿名言接荒诞下半句。「人类永远无法从历史中学到任何教训。但是只要你纠错数据给的够多，AI 祂是真学啊」。单字的选择（祂）可以顶一段话
- **风马牛串联**：把不相干的事物焊在一起，同时保持合理性与荒诞性（预制菜 × 组件库 × 「难吃的权利」）。串联的落点要回到读者自己身上
- **贴脸嘲讽**：反问轰炸（「AI 凭啥遵守 Spec，就因为你会啊啊啊啊战吼吗？」）+ 俗语化用（「让英雄去查英雄，让好汉去查好汉」）+ 把对方台词重复到荒诞（「系统不能有 bug！」×3 放进灾难现场）。每句嘲讽内部必须藏一个真论点，嘲讽对象是行为不是人群，抬一手再摔比全程踩更疼
- **喻体宇宙**：跨文章复用同一套比喻世界观（Markdown 皇帝 → 女皇帝 → 赛博佞臣 → 蛮族入侵），老读者会认，新读者不影响理解，系列感是免费的传播加成
- **预判反驳**：开头收缴读者的武器。「我知道 XX 孝子很多，先别急着骂，看完再说」
- **骂要有据，骂完给出路**：每段情绪输出后面跟着 issue 编号、源码文件名、链接；每个骂点附替代方案。情绪是钩子，证据是脊柱，洞见是尾巴——骂到最凶处突然拔高成真知，是最高级的骂战
- **敢于不解决**：提出问题不给答案，「我赌后者」三个字收尾。比强行升华高级得多

## 完稿自检

1. 挑出全文最工整的一段，把它弄脏（跑题 / 砍掉收尾 / 换成具体事实）
2. 数一下形容情绪的词，全部换成事实
3. 看结尾：是不是在升华？升华就砍，换成一个具体的、不甘心的细节（「那个鸡腿最后没吃成」）
4. 通读一遍问自己：读者能在哪一段看出「设计意图」？能看出来的地方就是要重写的地方
5. 真假难辨类：检查传奇与精确是否互相担保——每个离谱的宣称旁边都要站着一个过于扎实的数字
